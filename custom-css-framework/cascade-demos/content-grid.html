Content-Grid System

A powerful 7-column CSS Grid system for sophisticated content layouts with semantic breakout zones.

Grid Zones Explained

The content-grid creates five distinct zones for placing content. Each zone serves a specific purpose in your layout hierarchy.

Content (default)
.content--feature
.content--feature-max
.content--full
.content--full-safe
Content Zone (Default)
All direct children of .content-grid automatically span the content area. This is your readable content zone, typically around 65ch wide for optimal line length.
.content--feature
Extends beyond content width into the feature columns. Perfect for pull quotes, small callouts, or subtle emphasis without going full-width.
.content--feature-max
Extends further into the feature-max columns. Ideal for larger images, video embeds, or content that needs more visual prominence.
.content--full
Full-bleed edge to edge with no gutters. Use for hero images, immersive backgrounds, or when you want content to touch the viewport edges.
.content--full-safe
Full-width but maintains gutter padding. Best of both worlds - spans the full width while keeping comfortable spacing from viewport edges.

Notice how elements automatically return to content width. The grid flows naturally between different contexts without additional markup.

Practical Example: Article Layout

Here's how content-grid creates a professional article layout with mixed content widths.

Featured Image (feature-max)

The content-grid pattern elegantly solves a common design challenge: maintaining readable line lengths for text while allowing certain elements to break out for visual impact.

"The seven-column grid provides five distinct content zones, creating natural rhythm and visual interest in your layouts."

Unlike traditional container approaches that require wrapper divs and negative margins, content-grid uses CSS Grid's named lines to create semantic breakout zones. Simply add a class to any element to place it in a wider zone. 

<article class="content-grid">
  <h1>Article Title</h1>
  <p>Content stays readable...</p>

  <img class="content--feature-max" src="hero.jpg">

  <blockquote class="content--feature">
    Pull quote breaks out slightly...
  </blockquote>

  <div class="content--full">
    Full-bleed section...
  </div>
</article>

The grid automatically handles responsive behavior. On smaller screens, the breakout columns collapse gracefully while maintaining gutter spacing. No additional media queries needed.

Responsive Behavior

Resize your browser to see how the grid adapts:

  • Desktop (768px+): All zones display at full width
  • Tablet (600-768px): Feature-max collapses to feature width
  • Mobile (<600px): Feature zones collapse to content width
  • Small Mobile (<375px): Tighter gutters for maximum content space
